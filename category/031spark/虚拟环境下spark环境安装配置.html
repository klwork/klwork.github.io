虚拟环境下spark环境安装配置

大数据学习 Spark 2019/03/10

机器配置

假设已经成功配置了三台虚拟机器，分别为:master,node1,node2。
安装的都为centos7。

步骤

安装JDK开发插件

在master机器中

安装
yum install -y java-1.8.0-openjdk-devel

通过rpm -qa | grep openjdk查看OpenJDK版本

查看OpenJDK路径
whereis java

编辑 /etc/profile，添加的代码如下

export JAVA_HOME=/usr/lib/jvm/java-1.8.0-openjdk-1.8.0.201.b09-2.el7_6.x86_64

export CLASSPATH=.:$JAVA_HOME/jre/lib/rt.jar:$JAVA_HOME/lib/dt.jar:$JAVA_HOME/lib/tools.jar

export PATH=$PATH:$JAVA_HOME/bin

使配置生效
source /etc/profile

设定 ssh key

在master机器中

sudo su -
vim ~/.ssh/id_rsa.pub

复制其中的内容
自己的机器也需要进行复制
cat ~/.ssh/id_rsa.pub >> ~/.ssh/authorized_keys
在node1和node2机器中

vim ~/.ssh/authorized_keys

将上面复制的文件，放在其中。

在master机器中：
执行ssh node1，如果能够登录，则设置成功。

启动hdfs

hdfs namenode -format

$HADOOP_PREFIX/sbin/start-dfs.sh

成功会启动3个进程

4228 NameNode
4327 DataNode
4488 SecondaryNameNode

测试是否成功

使用 hdfs dfsadmin -report或者访问浏览器进行访问。
http://10.168.1.100:50070

启动yard

$HADOOP_PREFIX/sbin/start-yarn.sh

这时会多启动进程
4816 NodeManager
4713 ResourceManager

netstat -nlop

启动spark

进入到master机器，执行
$SPARK_HOME/sbin/start-all.sh

访问
http://10.168.1.100:8080/

启动进程如下
5175 Master
5244 Worker

启动hive

启动
nohup hive –service metastore &

pyspark测试

pyspark –master spark://10.168.1.100:7077
